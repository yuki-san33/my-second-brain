# プロンプト作成の基本コツ

## 概要
効果的なプロンプトを作成するための4つの基本原則。これらを意識するだけで、AIからの回答の質が大幅に向上する。

関連: [[Claude/claude-prompting]]では、プロンプト設計を「ゴール・文脈・制約の3点セット」として整理している。

## 重要ポイント

### 1. 役割を明確にする
**例**: 「あなたは〇〇の専門家です」

AIに特定の専門家の視点で回答してもらうことで、より専門的で信頼性の高い回答が得られる。

### 2. 具体的な指示を出す
**例**: 「〇〇について説明してください」

曖昧な質問ではなく、何を求めているかを明確にすることで、期待通りの回答が得られる。

実践例: [[prompt-examples/explanation]]（説明用プロンプト）、[[prompt-examples/debugging]]（デバッグ用プロンプト）

### 3. 出力形式を指定する
**例**: 「箇条書きで3つ」

回答の形式を事前に指定することで、後処理が楽になり、使いやすい結果が得られる。

実践例: [[prompt-examples/code-generation]]では、コード生成時にスタイルガイドと制約を明示する方法を解説している。

### 4. 制約を付ける
**例**: 「200文字以内で」

長さや範囲などの制約を設けることで、適切な量の情報が得られ、無駄を削減できる。

実践例: [[prompt-examples/code-generation]]では、コード生成時の制約の付け方を詳しく説明している。

## 実践アクション
- [ ] 新しいプロンプトを作成する際は、この4つの原則をチェックリストとして使用する
- [ ] 既存のプロンプトを見直し、改善できる点がないか確認する
- [ ] 効果的なプロンプトの例を収集し、パターン化する
- [ ] [[promptops]]の手法を活用して、プロンプトのバージョン管理と評価を行う
